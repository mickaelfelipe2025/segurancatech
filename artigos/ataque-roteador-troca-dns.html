Analisei um caso real de ataque a roteador com troca de DNS

Leitura: 24–30 min • Atualizado: 18/02/2026
Neste caso real, o cliente acessava o site do banco normalmente — mas era redirecionado para uma página falsa sem perceber.

Este incidente ocorreu com um cliente residencial que relatou dificuldades para acessar o aplicativo do banco após uma atualização do roteador. Segundo ele, ao tentar acessar o site da instituição financeira pelo navegador, a página carregava normalmente — inclusive com o cadeado HTTPS.

No entanto, após inserir login e senha, a página retornava um erro de autenticação. Minutos depois, o cliente recebeu uma ligação informando que havia sido realizada uma transferência via PIX que ele não reconhecia.

Investigação técnica

Durante a análise, foi identificado que o roteador do cliente havia sofrido alteração nas configurações de DNS. Em vez de utilizar servidores confiáveis, o equipamento estava configurado para utilizar um DNS malicioso.

Isso permitia que o tráfego destinado ao site do banco fosse redirecionado para um servidor controlado pelo criminoso.

Como o ataque ocorreu

  • Roteador com senha padrão
  • Acesso remoto habilitado
  • Firmware desatualizado
  • DNS alterado remotamente

Consequências

O cliente realizou login em uma página falsa acreditando estar no site oficial.

O cadeado HTTPS não impediu o golpe.

Primeiras 24h

  • Resetar o roteador
  • Alterar senha de acesso
  • Trocar DNS manualmente
  • Notificar banco

Plano de 7 dias

  • Atualizar firmware
  • Desativar acesso remoto
  • Monitorar contas

Prevenção

  • Trocar senha padrão
  • Atualizar firmware
  • Usar DNS confiável

Checklist

  • Senha forte
  • Firmware atualizado
  • Acesso remoto desativado
